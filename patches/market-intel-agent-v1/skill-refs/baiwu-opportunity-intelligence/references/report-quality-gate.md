# Report quality gate

This file defines the minimum acceptable quality bar for daily opportunity briefs.

## Hard failure patterns

The report is not acceptable if it contains any of these patterns:

- “未发现具体机会” with no coverage explanation
- macro statements like “预计增长” without evidence
- generic advice like “继续关注市场”
- no named institution, no branch, no region, and no source
- no mention of search coverage when results are sparse

## Minimum bar for each concrete lead

A lead should not appear in `今日高价值机会` or `值得跟进的次级线索`
unless it has most of the following:

- named buyer or institution
- branch or region
- project / procurement theme
- source or evidence clue
- fit to one of Baiwu's offerings
- next action

## Minimum bar for sparse-result days

If concrete leads are limited, the report must still include:

- searched account pools
- source types touched
- representative query families used
- what failed to return useful results
- where the next expansion should go

## Preferred output behavior

- fewer but real leads is better than many vague claims
- evidence beats prediction
- search-diagnostic value matters when lead volume is low
