Execution Flow

How a request flows through the platform

Every user request passes through all four layers in sequence — Console handles entry, Core manages state and policy, Studio loads agent definitions, Router selects the right model.

User Request → Console → Core creates task → Studio loads definitions → Policy check → Router selects LLM → Tool / MCP action → Timeline / Audit
Quick Install

Get started in minutes

Docker Compose

Quickest start

For evaluation, internal pilots, and small team deployments.

Shell 
curl -fsSL https://raw.githubusercontent.com/chenbinsg/goku-core/main/install.sh | bash
Helm + Kubernetes

Production scale

Full Kubernetes deployment with Helm for production environments.

Shell 
helm repo add goku https://chenbinsg.github.io/goku-core/helm
helm install goku goku/goku
Python SDK

Developer integration

Install the Python SDK to integrate Goku into your own applications.

Shell 
pip install goku-sdk

# TypeScript
npm install @goku/sdk
